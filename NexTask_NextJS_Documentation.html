NexTask Architecture & Technical Specifications

A Modern Full-Stack AI Productivity Application
Executive Summary: NexTask is an enterprise-grade task management system migrated from a Vanilla JavaScript prototype into a highly scalable, full-stack Next.js application. It leverages Cloud PostgreSQL (Supabase) for data persistence and Google's Gemini AI for natural language processing and automated categorization.

1. The Technology Stack

Technology Role in Project Why We Chose It (The "Why")
Next.js (React) Frontend Framework & UI Architecture React allows for highly modular, component-based UI development. Next.js was chosen specifically for its App Router, optimized build system, and industry-standard dominance. It drastically simplifies state management compared to Vanilla JS.
Supabase Backend-as-a-Service (BaaS) & Database Supabase provides an instant, secure PostgreSQL database with built-in authentication. It acts as an open-source alternative to Firebase, allowing us to implement secure user logins and persistent cloud storage without writing complex backend server code.
Google Gemini AI Artificial Intelligence Processing The Gemini 2.5 Flash model is incredibly fast and highly capable of natural language understanding. We use it to read the user's plain-text task and intelligently sort it into predefined categories.
Vanilla CSS Styling & Aesthetics To prove fundamental design competency, we utilized pure CSS with advanced techniques like Glassmorphism, CSS Grid, and animated mesh gradients rather than relying on heavy CSS frameworks like Bootstrap or Tailwind.

2. Core Features & How They Work

A. Secure Authentication Engine

NexTask utilizes Supabase Auth for complete identity management. Instead of building a custom JWT system, we offload security to Supabase. When a user logs in, Supabase issues a secure session token. In our code, we use React's useEffect hook to listen to the onAuthStateChange event, ensuring the UI immediately reflects the user's logged-in status and securely fetches only their specific tasks from the database using Row Level Security (RLS).

B. AI-Powered Task Categorization

When a user submits a new task (e.g., "Finish the React tutorial video"), the application pauses the submission and sends the text string securely to the Google GenAI API. The system prompt forces the AI to respond with a single word (Coding, Coursework, Personal, or Other). This eliminates manual sorting for the user, creating a frictionless UX.

C. Base64 Multi-File Uploads

Users can attach up to 5 files per task (Max 5MB each). To simplify database architecture, we use the HTML5 FileReader API to convert physical files into Base64 Strings. These strings are stored directly inside a JSON array in the Supabase PostgreSQL database, ensuring all text and file data is synced perfectly without needing complex external Storage Buckets.

D. Client-Side Background Alarms

A background loop runs continuously via a setInterval inside a React useEffect hook. It compares the current system time against the database timestamps of active tasks. If a task is within 6 hours of its deadline, the state engine triggers a browser alert. It updates the database so the user is only warned once.

3. Deep Dive: Understanding the Codebase

If you are reviewing this code for an interview, the most critical file is app/page.js. Here is a breakdown of the advanced React concepts utilized:

State Management (React Hooks)

Instead of manually manipulating the DOM with document.getElementById, we use the useState hook. When data changes, React automatically repaints the UI.

// We store all tasks in an array. // Calling setTasks() instantly updates the screen. const [tasks, setTasks] = useState([]); const [newTaskTitle, setNewTaskTitle] = useState('');

Asynchronous Data Fetching

Because database reads and AI requests take time, we use modern ES6 async/await syntax to ensure the code executes sequentially without blocking the browser thread.

const loadTasks = async () => { // Wait for Supabase to return the user's data const { data, error } = await supabaseClient .from('tasks') .select('*') .order('created_at', { ascending: false }); // Update the React State with the new data if (data) setTasks(data); };

4. Project Folder Structure

The repository adheres to the strict Next.js App Router conventions:

  • app/page.js — The main React component containing all business logic, UI state, and rendering.
  • app/layout.js — Handles the global HTML document structure and Next.js metadata.
  • app/globals.css — The global design system containing all CSS variables and classes.
  • lib/supabaseClient.js — The initialization code connecting our app to the PostgreSQL backend.
  • package.json — The Node.js dependency manifest containing the exact versions of React, Next, and Supabase required to run the project.

5. Running the Project Locally

Because this is a Node.js project, running it requires a local Node environment.

  1. Download the project from GitHub and open a terminal in the project root.
  2. Run npm install to download all required dependencies listed in package.json.
  3. Run npm run dev to start the local development server.
  4. Navigate to http://localhost:3000 in any web browser.
